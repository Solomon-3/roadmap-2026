API
an API is simply a server exposing structured routes like: GET /users, POST /users

flow 
client request -> server -> routecheck -> /users,/products,other
browser (GET /usess) -> node server(checks: req.url === "/usess", req.method === "GET") -> Route matched -> server sends JSON["alice","bob","solo"] 

servers send JSON 
json is the universal language for APIs
'JSON.stringify(users)'

now lets upgrade our server to also accept data ie POST
curl -X POST http://localhost:3000/users -d "Spencer"

a) so when yo run that command , the tool curl sent this http request:
POST /users
Body: Spencer

b) your Node.js server then executed this flow:
Client(curl) (POST /users, Body:Spencer) -> Node server (req.method ===POST, req.url ===/users) -> server starts reading body (req.on("data")) -> Body collected (req.on("end")) -> users.push("Spencer") -> response sent("User added")


we have a weakness you are sending "spencer" as plain text yet real real APIs send it as structured JSON
{
   "name": "Spencer"
}

curl -X POST http://localhost:3000/users \
-H "Content-Type: application/json" \
-d '{"name":"xuusd"}'

up to now your server receives requests, checks routes, sends responses
today we learn about middlewarethinking (request logging)

middleware means:
request comes in -> somethins runs BEFORE the route -> route logic runs -> response goes back
eg : logging requests, authentication, request validation, rate limiting

today we build request logging which is the simplest
after running the server and opening local host then check your terminal

flow
Browser -> incoming request -> logging layer -> route logic -> response
this will b so useful when you start express.js
without logs, debugging becomes impossible

flow
request -> console.log -> route logic -> response
request -> logger(req) -> route logic -> response
in express it lookslike this: 'app.use(logger)'

middleware handles things like authentication, request validataion, logging, security headers, rate limiting

flow
request arrives -> logger(req) runs first -> route condition check -> /about route matched -> "about route accessed"prints -> responsesent to browser
